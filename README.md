# autoAutodock
sequential docking + GUI

## Основные файлы для запуска докинга:
1. list_files.py - собирает доступные дочерские файлы из папки
2. real_auto_dock.py - взаимодействует с Виной, доступны 3 вида докинга
3. AD_Vina_GUI.py - графический интерфейс (когда-нибудь сделаю его более гибким)
Их нужно сохранить в папку, из которой будет осуществляться докинг (т.е. там же должны лежать белки, лиганды, конфиги)

## Как работать
1. В файле real_auto_dock.py проверить путь к файлам докинга и к самой Вине: строчки 15, 31, 78
2. В файле AD_Vina_GUI.py указать нужный тип докинга - найти 9 строчку и вписать нужную переменную: 
   ```
   dock_type = 'simple'
   dock_type = 'blind'
   dock_type = 'double'
   ```
3. Запустить AD_Vina_GUI.py
4. Если нужно дочить не всё подряд, ввести названия белков и лигандов в соответствующие поля. Использовать названия В ТОЧНОСТИ как у файлов, но БЕЗ .pdbqt (то есть только шифры)
5. Если нужно дочить всё подряд, нажать кнопку list all names
6. Можно комбинировать подходы: указать несколько конкретных лигандов и собрать все белки (в обратную сторону тоже сделаю)
7. Нажать Dock!
8. В папке со скриптами и белками будут появляться соответствующие логи и total_nrg.txt, который станет доступен по завершении процесса. В этом файле - энергия первой позиции каждого лиганда с каждым белком
