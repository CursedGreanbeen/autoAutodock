# autoAutodock
sequential docking + GUI

## Основные файлы для запуска докинга
Их нужно сохранить в папку, из которой будет осуществляться докинг (т.е. там же должны лежать белки, лиганды, конфиги)
1. list_files.py - собирает доступные дочерские файлы из папки
2. real_auto_dock.py - взаимодействует с Виной, доступны 3 вида докинга
3. AD_Vina_GUI.py - графический интерфейс (когда-нибудь сделаю его более гибким)

## Как работать
1. В файле real_auto_dock.py проверить путь к файлам докинга и к самой Вине: строчки 15, 31, 42, 78; аналогично - в файле list_files.py (строка 4)
2. В файле AD_Vina_GUI.py указать нужный тип докинга - найти 9 строчку и вписать нужную переменную: 
   ```
   dock_type = 'simple'
   dock_type = 'blind'
   dock_type = 'double'
   ```
3. Запустить AD_Vina_GUI.py
4. Если нужно дочить не всё подряд, ввести названия белков и лигандов в соответствующие поля. Использовать названия В ТОЧНОСТИ как у файлов, но БЕЗ .pdbqt (то есть только шифры)
5. Если нужно дочить всё подряд, нажать кнопку list all names
6. Можно комбинировать подходы: указать несколько конкретных лигандов и собрать все белки (в обратную сторону тоже сделаю)
7. Нажать Dock!
8. В папке со скриптами и белками будут появляться соответствующие логи и total_nrg.txt, который станет доступен по завершении процесса. В total будет энергия первой позиции каждого лиганда с каждым белком (то есть лучшие энергии связывания)

## Дополнительные файлы, которые не обязательны для последовательного автоматизированного докинга, но могут быть удобны на других этапах работы:
Для работы понадобится установить пакеты Python: rdkit, openbabel (в командной строке через pip install *название пакета*)
1. gather_nrg.py - очистка файла энергии для последующей вставки в эксель
2. grid_file_to_config.py - приведение стандартного грид-файла из AutoDock Tools в необходимый для докинга формат. В строке 61 нужно указать свои шифры белков
3. minimize_optimize.py - получение 3Д из SMILES. Для этого нужно вписать в предпоследнюю строку свои молекулы в формате: 
['Yohimbine COC(=O)[C@H]1[C@H](CC[C@@H]2[C@@H]1C[C@H]3C4=C(CCN3C2)C5=CC=CC=C5N4)O', 'Next_mol SMILESSMILESSMILES', ...]
4. get_pdbqt.py - получение .pdbqt из .sdf
